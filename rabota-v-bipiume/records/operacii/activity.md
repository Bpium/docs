# История

\[Скриншот: вкладка История — лента изменений]

### Как читать ленту

Изменения отображаются в хронологическом порядке — самые свежие сверху. Каждое событие содержит имя сотрудника, дату и время, а также перечень изменённых полей.

Для большинства изменений видно что было и что стало:

* **Было → Стало** — если значение изменилось
* **Только новое значение** — для некоторых типов полей

Каждая новая дата отделяется горизонтальной линией — удобно ориентироваться в длинной ленте. Комментарии из чата также отображаются в ленте истории и выделены фоном.

\[Скриншот: пример изменения в ленте — поле Статус, Было → Стало]\
\[Скриншот: разделительная линия между датами в ленте]

### Фильтрация ленты

Ленту можно фильтровать чтобы найти нужные изменения:

* **По сотруднику** — видеть только действия конкретного человека
* **По полям** — видеть изменения только выбранных полей

\[Скриншот: фильтр по сотруднику — выпадающий список]\
\[Скриншот: фильтр по отображаемым полям]

{% hint style="info" %}
История учитывает права доступа. Вы увидите только те изменения, к которым у вас есть доступ. Если администратор ограничил видимость определённых полей — изменения по ним в истории не отобразятся.
{% endhint %}

#### Просмотр истории изменений&#x20;

Внутри каждой записи есть вкладка с иконкой часов, которая содержит ленту истории изменений записи.  История изменений записи  позволяет просмотреть действия сотрудников по конкретной записи за все время: кто, что и когда делал.

<figure><img src="../../../.gitbook/assets/image (64).png" alt=""><figcaption></figcaption></figure>



Действия отсортированы по дате изменений: недавние — сверху, старые — снизу.

Система отображает изменения в следующих форматах:

1. Прежние и новые значения показаны в виде изменения: Было → Стало
2. Показаны только новые значения

В левой колонке указывается название записи. Сотрудники указаны справа от записи или отдельной строкой в записи как Исполнители. Каждая новая дата выделяется линией для удобной ориентации в ленте.

&#x20;В общей ленте активности также показаны комментарии, оставленные сотрудникам к записям.

## Фильтр изменений

Ленту активности можно фильтровать по:

* Сотрудникам: все сотрудники, конкретный сотрудник

* Отображаемым полям (видимость): можно выбрать отображение всех полей или конкретных

### Права

История учитывает права доступа к данным. Сотрудник увидит изменения только по тем действиям, которые ему доступны. Например можно задать настройки видимости таким образом, что сотрудник сможет видеть только свои изменения или не сможет видеть какие-либо изменения вовсе.
