# События

### Как это работает

Каждая запись в каталоге «События» — это одна подписка. В ней указано: за каким каталогом следить, какие именно действия отслеживать, и какой сценарий запускать в ответ.

Например: при изменении статуса в каталоге «Заявки» — запустить сценарий «Отправить уведомление менеджеру».

## Свойства

### Статус

Активно — подписка работает, при срабатывании события сценарий запускается.

Выключено — подписка приостановлена, события не обрабатываются. Используйте для временного отключения автоматизации без её удаления.

### Название и описание

Произвольное название и комментарий для идентификации подписки в каталоге. Используйте понятные названия — при большом количестве событий это упрощает навигацию.

### Каталог

Каталог, за изменениями в котором будет следить подписка.

### События

Типы действий, при которых срабатывает подписка. Подробнее — в разделе «Типы событий» ниже.

### Отслеживать поля

Список ID полей, изменение которых будет запускать подписку. Если оставить пустым — подписка будет срабатывать при изменении любого поля записи.

{% hint style="info" %}
ID поля можно посмотреть в конструкторе каталога в разделе «Технические свойства» панели свойств поля.
{% endhint %}

### Выполнить

Действие, которое будет выполнено при срабатывании события из подписки. Возможные варианты: запуск процесса по [сценарию](scripts.md).

### Типы событий

Все события делятся на три вида — в зависимости от того, когда они срабатывают относительно операции.

#### Полный список событий

* Уведомление о создании записи — срабатывает после создания записи
* Уведомление об изменении записи — срабатывает после изменения записи
* Уведомление об удалении записи — срабатывает после удаления записи
* Запрос на создание записи — срабатывает до создания, ожидает одобрения сценария
* Запрос на изменение записи — срабатывает до изменения, ожидает одобрения сценария
* Запрос на удаление записи — срабатывает до удаления, ожидает одобрения сценария
* Изменено поле во время редактирования — срабатывает после изменения поля, до сохранения
