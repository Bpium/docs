# Сценарии

## Как сценарий связан с событием

Сам по себе сценарий не запускается — он привязывается к событию. В каталоге «События» создается подписка, в которой указывается: за каким каталогом следить и какой сценарий запускать при срабатывании. Одному сценарию может соответствовать несколько событий.

Подробнее о настройке подписок читайте в статье [События](events.md).

## Свойства

### Название

Обязательное поле. Используется для выбора сценария в каталоге «События» при настройке подписки. Название должно быть понятным — по нему администратор находит нужный сценарий в списке.

### Описание

Произвольный комментарий к сценарию — для чего он нужен, что делает, где используется.

### Сценарий

Файл со схемой автоматизации в формате <mark style="color:green;">\*.bpmn</mark>. Создается и редактируется в визуальном редакторе сценариев — он открывается при нажатии в поле «Сценарий» на кнопку «Создать».\
Также можно загрузить готовый сценарий в систему с компьютера или прикрепить по ссылке.

Готовый файл <mark style="color:green;">\*.bpmn</mark> можно скачать из карточки и загрузить в другую систему Бипиума — это удобно для переноса автоматизаций между окружениями.

{% hint style="info" %}
Редактор сценариев — отдельный инструмент для построения автоматизаций. Подробнее о компонентах, переменных и логике читайте в статье [Автоматизации](../../processes/).
{% endhint %}
