# Версии и обновления

## v3.0.0 от 30 января 2026

Версия API: 3.0.0\
Версия веб-приложения: 3.0.1\
Версия сервера процессов: 3.0.9<br>

**Ключевые изменения:**

* Добавлены формулы: значение поля записи может автоматически вычисляться по формуле.
* Добавлено ИИ поле: значение поля записи может автоматически получать ответ от моделей ИИ.
* Добавлен компонент сценариев искусственного интеллекта, позволяющий делать запрос к разным ИИ моделям (DeepSeek, ChatGPT, Grok, Sonar/Perplexity, YandexGPT, GigaChat).
* Добавлен компонент сценариев автоматической отправки сообщений в Telegram с возможностью отправки от имени пользователя или бота.
* Добавлены зависимости в настройках каталога: можно увидеть, где логика работы связей может нарушиться при изменении / удалении поля.
* Добавлена авторизация через публичные сервисы (Яндекс, Google, ВК, и другие)
* Добавлена авторизация через внутрикорпоративные сервисы на базе OAuth2.0, OpenID, KeyCloak, SSO
* Добавлен скрытый тип поля в карточку записи, чтобы пользователи не видели лишнюю информацию.
* Добавлена вкладка «Инструкция», чтобы создатель каталога мог подробно описать заложенные бизнес-процессы и правила работы с каталогом.<br>

**Ключевые изменения в веб-приложении:**

* Добавлены кнопки «Сохранить» и «Не сохранять»: при закрытии записи теперь появляется удобный выбор действия.
* Добавлено отображение текстового редактора, связанных записей в режиме таблицы и карточки во всю ширину — они автоматически растягиваются под размер секции / вкладки.
* Изменилась форма боковой панели в карточке записи: она значительно расширена, добавлено отображение связей, истории изменений и вкладка с инструкцией.
* Улучшены индикаторы свойств полей в настройках каталога (условная видимость, нередактируемая, скрытая, наличие событий).
* Исправлена логика доступа к импорту записей: пользователь получает доступ к импорту, если хотя бы один из его правовых видов позволяет создавать записи.
* Исправлена ошибка отображения событий в календаре, у которых не указана дата окончания.<br>

**Ключевые изменения в сервере API:**

* Оптимизирована скорость работы за счет повышения производительности структуры базы данных и кеширования видов каталогов.
* Исправлена ошибка в правовой политике: при изменении ID поля права доступа, назначенные ранее на это поле, переставали работать.
* Исправлена ошибка прерывания процессов: ранее при остановке процесса сервер некорректно устанавливал статус.
* Исправлена ошибка авторизации через Active Directory: ранее система система запрещала вход пользователя, даже если авторизационные данные были верны.
* Исправлена ошибка присвоения ID новым полям: при создании нового поля заданный пользователем ID не сохранялся.
* Исправлена проблема с кэшированием при обновлении полей каталога.
* Исправлена ситуация, когда сообщения могли не загружаться.
* Исправлена ошибка отображения времени создания в истории записи.
* Исправлена ошибка добавления нового поля в системный каталог.<br>

**Ключевые изменения в сервере процессов:**

* Добавлена возможность конвертации документов из DOCX в PDF.
* Добавлена логика ограничения числа активных сценариев по доменам, чтобы предотвратить перегрузку серверов процессов в облаке.
* Добавлена возможность размножения диапазонов строк в компоненте Excel с поддержкой вложенных массивов (если внутри этих строк есть разные массивы, они тоже размножаются внутри каждого повторённого диапазона) для создания иерархических отчётов и сложных документов.
* Оптимизирована генерация Excel-файлов: десятки тысяч строк обрабатываются за 1–2 секунды вместо нескольких минут.&#x20;
* Исправлено копирование формул при размножении строк в Excel-файлах.
* Исправлено зависание в компоненте генерации Excel, возникающее если в данных забыть передать значения, используемые в Excel-шаблоне. Теперь вместо забытых данных подставляются пустые строки.
